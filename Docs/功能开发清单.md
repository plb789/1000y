## 项目架构概述

### 技术栈

**前端（Electron客户端）**：
- 原生HTML + Canvas 2D + JavaScript
- WebSocket长连接通信
- 兼容原版千年资源格式（DDS贴图、SPR帧动画、MAP地图）

**服务端（Go微服务）**：
| 服务 | 端口 | 职责 |
|------|------|------|
| GatewayService | 8080 | WebSocket连接接入、消息转发、广播 |
| LoginService | 8081 | 账号校验、登录授权、Token发放 |
| GameService | 8082 | 角色、战斗、武学、场景核心逻辑 |
| DBService | 8083 | 数据库CRUD、缓存管理、服务注册 |

## 开发优先级建议

### 第一阶段（核心玩法）
1. 战斗系统（攻击判定、伤害计算、怪物AI）
2. 武学系统（技能释放、冷却管理）
3. 道具系统（背包、装备穿戴）
4. 角色面板与状态栏UI

### 第二阶段（社交玩法）
1. 任务系统（接取、追踪、奖励）
2. 好友系统
3. 组队系统
4. 门派系统

### 第三阶段（扩展玩法）
1. 交易系统（面对面交易、摆摊）
2. 活动系统
3. 商城系统
4. 音效系统
