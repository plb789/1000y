# 🎮 怪物生成系统 - 使用指南

## 📋 功能概述

已为**《千年江湖》**添加完整的**自动怪物生成系统**，包括：

✅ **启动时自动生成** - 服务端启动时为每个地图生成初始怪物

## 🚀 自动生成机制

### 生成时机
服务端启动时 (`main.go` → `loadManagedMaps()` 之后)

### 生成规则

| 怪物类型 | 生成数量 | 说明 |
|----------|----------|------|
| 普通怪 (type=0) | 3-8个/模板 | 随机分布 |
| 精英怪 (type=1) | 1-2个/模板 | 较少出现 |
| BOSS (type=2) | 1个/模板 | 唯一存在 |

### 生成位置
- 坐标范围: `(50, 50)` 到 `(950, 950)`
- 避开地图边缘和出生点中心区域
- 完全随机分布

## 🔧 GM命令接口（HTTP API）

### 接口列表

#### 1. 手动生成怪物
```
POST http://localhost:8080/api/monster/gm/spawn
Content-Type: application/json
```

**请求参数：**
```json
{
    "base_id": 101,      // 必填 - 怪物模板ID（参考monsters.json）
    "map_id": 1,         // 必填 - 目标地图ID
    "x": 100,            // 可选 - X坐标（0=随机）
    "y": 200,            // 可选 - Y坐标（0=随机）
    "count": 5           // 可选 - 生成数量（默认1，最大20）
}
```

**响应示例：**
```json
{
    "code": 200,
    "msg": "成功生成5个怪物",
    "data": [
        {
            "id": 1001,
            "base_id": 101,
            "name": "野鸡",
            "level": 1,
            "x": 100,
            "y": 200,
            "hp": 50,
            "max_hp": 50,
            "status": 0
        },
        // ... 更多怪物
    ]
}
```

**使用示例：**

```bash
# 在坐标(500, 500)生成10只野鸡
curl -X POST http://localhost:8080/api/monster/gm/spawn \
  -H "Content-Type: application/json" \
  -d '{"base_id": 101, "map_id": 1, "x": 500, "y": 500, "count": 10}'

# 在随机位置生成1只BOSS（山贼头目）
curl -X POST http://localhost:8080/api/monster/gm/spawn \
  -H "Content-Type: application/json" \
  -d '{"base_id": 204, "map_id": 2}'

# 批量生成5只精英狼
curl -X POST http://localhost:8080/api/monster/gm/spawn \
  -H "Content-Type: application/json" \
  -d '{"base_id": 301, "map_id": 2, "count": 5}'
```

## 🔍 故障排查

### 问题1：启动后没有看到怪物

**检查步骤：**
```bash
# 1. 查看服务端日志
grep "初始化完成" logs/gameservice.log
# 应该输出: ✅ 地图1初始化完成: 共生成XX个怪物实例

# 2. 检查配置是否加载
grep "怪物" logs/gameservice.log | head -5
# 应该输出: 配置加载完成: ... 19个怪物 ...

# 3. 手动查询API
curl http://localhost:8080/api/monster/list/1
# 如果返回空数组，说明生成失败
```

**可能原因及解决：**
- ❌ `common.GameConfig.Monsters` 为空 → JSON文件路径错误或格式错误
- ❌ `globalService` 为nil → `InitService()` 未被调用或调用顺序错误
- ❌ `rand.Intn` panic → 随机数种子未初始化（Go 1.20+自动处理）

**解决方案：**
```go
// 在main.go开头添加：
// rand.Seed(time.Now().UnixNano()) // Go 1.20之前需要
```

---

### 问题2：GM命令返回错误

**错误信息：** `"怪物服务未初始化"`

**原因：** `InitService()` 尚未执行完毕就调用了GM接口

**解决：** 等待服务完全启动后再调用GM接口

---

**错误信息：** `"生成失败: 怪物不存在"`

**原因：** `base_id` 不存在于 `monsters.json`

**解决：**
```bash
# 查看可用的怪物ID
cat Server/GameService/Config/monsters.json | grep '"id"'
```

---

**错误信息：** HTTP 400 Bad Request

**原因：** 参数格式错误或缺少必填字段

**解决：** 检查JSON格式，确保包含 `base_id` 和 `map_id`

---

### 问题3：生成了但看不到

**检查步骤：**
```bash
# 1. 确认怪物确实生成了
curl http://localhost:8080/api/monster/list/1 | python -m json.tool

# 2. 检查坐标是否合理
# X/Y应该在 50-950 范围内

# 3. 检查前端是否正确接收怪物列表
# 打开浏览器F12 → Network → WS标签
# 查找 monster_list 消息
```

**可能原因：**
- ❌ 坐标超出可视范围（玩家看不到那么远）
- ❌ 前端未订阅怪物列表更新
- ❌ WebSocket连接断开

**解决：**
```javascript
// 在Game.js中添加手动刷新：
setInterval(() => {
  window.GameWS.send(Protocol.CMD_GET_MONSTER_LIST, {map_id: 1});
}, 5000); // 每5秒请求一次
```

```bash
#!/bin/bash
# monitor_monsters.sh - 怪物监控脚本

while true; do

  echo "=== 怪物监控系统 ==="

  echo "按 Ctrl+C 退出"

done
```

## 🎓 进阶用法

### 自定义生成位置

如果想在特定位置生成怪物（如任务点、副本入口），修改 `InitMapMonsters()` 的坐标逻辑：

```go
// 示例：在固定位置生成BOSS
if monsterConfig.Type == 2 { // BOSS
    x = 800  // 固定X坐标
    y = 800  // 固定Y坐标
}

// 示例：围绕出生点环形分布
angle := float64(i) / float64(count) * 2 * math.Pi
radius := 200.0 + rand.Float64() * 100
x = 500 + int(radius*math.Cos(angle))
y = 500 + int(radius*math.Sin(angle))
```

### 定时补充怪物

添加定时器，当怪物数量低于阈值时自动补充：

```go
// 在 InitService() 中启动goroutine
go func() {
    ticker := time.NewTicker(60 * time.Second) // 每60秒检查一次
    defer ticker.Stop()
    
    for range ticker.C {
        for _, mapID := range handledMaps {
            monsters := globalService.GetMonstersByMap(mapID)
            if len(monsters) < 10 { // 少于10个就补充
                log.Printf("地图%d怪物不足，自动补充", mapID)
                InitMapMonsters(mapID)
            }
        }
    }
}()
```
