# 📋 怪物生成配置文件 - 完整使用指南

## 🎯 概述

**配置文件**: `Server/GameService/Config/monster_spawns.json`  
**版本**: v1.0  
**用途**: 精确控制每个地图的怪物生成位置、数量、行为等

### 核心优势

✅ **精确控制** - 每个怪物的坐标、数量、半径都可配置  
✅ **灵活开关** - 单个生成点可独立启用/禁用  
✅ **热更新** - 无需重启服务即可修改配置  
✅ **可视化编辑** - 支持从运行状态导出为配置  
✅ **Fallback机制** - 配置缺失时自动使用算法生成  

---

## 📁 文件结构

```json
{
  "version": "1.0",                    // 版本号
  "description": "描述信息",
  "last_updated": "2026-06-20",
  "maps": {
    "1": { ... },                      // 地图1的配置
    "2": { ... },                      // 地图2的配置
    "3": { ... }                       // 地图3的配置
  }
}
```

---

## 🔧 配置字段详解

### 顶层字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `version` | string | 是 | 配置格式版本（目前固定"1.0"） |
| `description` | string | 否 | 配置描述说明 |
| `last_updated` | string | 否 | 最后更新时间 |
| `maps` | object | 是 | 地图配置集合（key=地图ID字符串） |

---

### 地图级配置 (MapSpawnConfig)

每个地图的完整配置：

```json
{
  "map_id": 1,
  "map_name": "新手村",
  "spawn_points": [ /* 生成点列表 */ ],
  "global_settings": { /* 全局设置 */ }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `map_id` | uint32 | 是 | 地图ID（与maps.json对应） |
| `map_name` | string | 否 | 地图名称（用于日志显示） |
| `spawn_points` | array | 是 | 该地图的所有生成点 |
| `global_settings` | object | 否 | 地图全局设置（可选） |

---

### 全局设置 (MapSpawnGlobalSettings)

```json
{
  "max_monsters_per_map": 50,     // 地图最大怪物数
  "auto_respawn": true,            // 是否自动复活
  "respawn_check_interval": 10,    // 复活检查间隔(秒)
  "despawn_distance": 500          // 玩家离开多远后消失(格)
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `max_monsters_per_map` | int | 100 | 该地图同时存在的最大怪物数量上限 |
| `auto_respawn` | bool | true | 怪物死亡后是否自动定时复活 |
| `respawn_check_interval` | int | 10 | 复活检查频率（秒） |
| `despawn_distance` | int | 500 | 当所有玩家离开此距离后，怪物消失回收资源 |

---

### 生成点配置 (SpawnPointConfig) - **核心**

这是最重要的部分，每个对象代表一个"怪物刷新点"：

```json
{
  "id": 1001,
  "name": "新手村-东门野鸡区",
  "base_monster_id": 101,
  "monster_name": "野鸡",
  "x": 650,
  "y": 500,
  "count": 3,
  "respawn_time": 30,
  "spawn_radius": 2,
  "level_range": [1, 3],
  "ai_type_override": null,
  "is_active": true,
  "description": "新手村东门附近，适合1-3级玩家"
}
```

#### 完整字段说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| **`id`** | uint32 | **是** | - | 生成点唯一ID（全局唯一，建议从1001开始） |
| **`name`** | string | **是** | - | 生成点名称（用于日志和调试） |
| **`base_monster_id`** | uint32 | **是** | - | 怪物模板ID（对应monsters.json的id） |
| `monster_name` | string | 否 | 自动读取 | 怪物名称（可选，方便阅读） |
| **`x`** | int | **是** | - | 中心X坐标（格） |
| **`y`** | int | **是** | - | 中心Y坐标（格） |
| **`count`** | int | **是** | 1 | 同时生成的最大数量 |
| `respawn_time` | int | 否 | 使用模板值 | 死亡后复活时间（秒） |
| **`spawn_radius`** | int | 否 | 0 | **生成半径（格）**：0=精确位置，>0=在圆内随机分布 |
| `level_range` | array[2] | 否 | [1,99] | 适用玩家等级范围 [最小, 最大] |
| `ai_type_override` | uint8/null | 否 | null | AI类型覆盖：null=默认, 0=被动, 1=主动, 2=BOSS |
| **`is_active`** | bool | **否** | **true** | 是否激活（可快速开关） |
| `description` | string | 否 | "" | 详细描述说明 |

---

## 💡 关键概念详解

### 1. spawn_radius（生成半径）- **重要！**

这个参数决定怪物如何在生成点周围分布：

#### 场景A：精确位置（spawn_radius = 0）
```json
{
  "x": 500,
  "y": 300,
  "spawn_radius": 0
}
```
**效果**: 所有怪物都精确生成在(500, 300)这个点  
**适用**: BOSS、任务NPC、固定守卫

#### 场景B：小范围分散（spawn_radius = 2-5）
```json
{
  "x": 650,
  "y": 500,
  "spawn_radius": 3
}
```
**效果**: 怪物在以(650,500)为中心、半径3格的圆内随机分布  
**适用**: 小型怪物群（野鸡、兔子等）

#### 场景C：大范围区域（spawn_radius = 10-20）
```json
{
  "x": 900,
  "y": 600,
  "spawn_radius": 15
}
```
**效果**: 怪物分散在较大区域内，看起来更自然  
**适用**: 大型怪物群（山贼营地、野外区域）

**推荐值**:
- BOSS/精英: `0` （固定位置）
- 小怪群: `2-5` 格
- 中型区域: `8-12` 格
- 大型区域: `15-25` 格

---

### 2. is_active（激活开关）

快速启用/禁用某个生成点，无需删除配置：

```json
// 启用状态（正常生成）
"is_active": true

// 禁用状态（不生成，但保留配置）
"is_active": false
```

**使用场景**:
- 节日活动：临时关闭某些怪物
- 版本更新：预先配置新内容但不激活
- 调试测试：单独测试某个生成点
- 维护模式：关闭危险BOSS

---

### 3. ai_type_override（AI类型覆盖）

临时修改怪物的AI行为，不影响原始模板：

```json
// 不覆盖（使用monsters.json中的ai_type）
"ai_type_override": null

// 强制改为被动型（即使原模板是主动）
"ai_type_override": 0

// 强制改为主动型
"ai_type_override": 1

// 强制改为BOSS型（永不逃跑）
"ai_type_override": 2
```

**典型用法**:
- 副本BOSS: 覆盖为类型2（确保不逃跑）
- 练功假人: 覆盖为类型0（只反击不追击）
- 特殊事件怪: 临时改变行为模式

---

### 4. level_range（等级范围）

标注该生成点适合什么等级的玩家（仅作标记，不影响实际生成）：

```json
// 新手区
"level_range": [1, 5]

// 中级挑战区
"level_range": [10, 20]

// 高级区域
"level_range": [30, 50]

// 全等级（无限制）
"level_range": [1, 99]
```

**作用**:
- 前端显示提示："该区域建议等级10-20级"
- 任务系统检查玩家是否达到要求
- 日志统计不同等级区域的活跃度

## 🛠️ 操作指南

### 方式1：直接编辑JSON文件

1. 打开 `Server/GameService/Config/monster_spawns.json`
2. 修改或添加配置
3. 保存文件
4. **热更新**（可选）：调用API或重启服务

**优点**: 可批量修改、版本控制（Git）  
**缺点**: 需要了解JSON格式

---

### 方式2：使用GM命令API（推荐）

#### 2.1 查看当前配置

```bash
# 查询地图1的生成点配置
curl http://localhost:8080/api/monster/spawn-config/1 | python -m json.tool

# 查询所有地图配置
curl http://localhost:8080/api/monster/spawn-config/all | python -m json.tool
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "map_id": 1,
    "map_name": "新手村",
    "spawn_points": [
      {
        "id": 1001,
        "name": "东门野鸡群",
        "base_monster_id": 101,
        "x": 650,
        "y": 500,
        "count": 3,
        "spawn_radius": 3,
        "is_active": true
      }
      // ... 更多
    ]
  }
}
```

---

#### 2.2 手动生成怪物（即时生效）

```bash
# 在指定位置生成怪物
curl -X POST http://localhost:8080/api/monster/gm/spawn \
  -H "Content-Type: application/json" \
  -d '{
    "base_id": 104,
    "map_id": 1,
    "x": 700,
    "y": 400,
    "count": 5
  }'
```

**场景**: 你用GM命令调整好怪物分布后，想保存为永久配置

1. 将输出的 `data` 部分复制到 `monster_spawns.json`

4. 保存文件

---

#### 2.4 热重载配置（无需重启！）

修改完 `monster_spawns.json` 后，调用热更新接口：

```bash
# 重新加载所有配置（包括monster_spawns.json）
curl -X POST http://localhost:8080/api/monster/reload-spawn-config
```

**响应**:
```json
{
  "code": 200,
  "msg": "✅ 配置热更新成功！下次生成怪物将使用新配置",
  "data": {
    "reload_time": "2026-06-20 15:30:45",
    "maps_count": 3
  }
}
```

**注意**: 
- ✅ 已存在的怪物不会消失或移动
- ✅ 新生成的怪物将使用新配置
- ✅ 如果想立即应用，需要手动清除现有怪物（重启或调用清理接口）

## 🎨 最佳实践

### 1. 命名规范

**生成点命名格式**: `{区域}-{怪物类型}`

好的例子：
```
✅ "东门-野鸡群"
✅ "北郊-野猪林"
✅ "山谷入口-恶狼巡逻队"
✅ "⭐BOSS-山贼头目寨"
✅ "副本入口-守卫"
```

不好的例子：
```
❌ "生成点1"
❌ "abc"
❌ "怪物A"
```

---

### 2. ID分配规则

避免ID冲突：

| 范围 | 用途 | 示例 |
|------|------|------|
| 1-999 | 系统保留 | - |
| 1001-1999 | 地图1的生成点 | 1001, 1002... |
| 2001-2999 | 地图2的生成点 | 2001, 2002... |
| 3001-3999 | 地图3的生成点 | 3001, 3002... |
| ... | 以此类推 | |

**公式**: `地图ID * 1000 + 序号`

---

### 3. spawn_radius选择指南

| 场景 | 推荐半径 | 原因 |
|------|----------|------|
| BOSS/精英 | 0格 | 固定位置，玩家主动寻找 |
| 任务NPC | 0格 | 精确位置，方便找到 |
| 小怪群（<5个） | 2-5格 | 小范围分散，自然但不散乱 |
| 区域怪物（5-15个） | 8-15格 | 较大区域，模拟野生分布 |
| 大型区域（>15个） | 15-25格 | 广阔地带，探索感强 |
| 巡逻路线 | 0格 + AI巡逻 | 由AI系统控制移动 |

---

### 4. 数量平衡建议

根据地图大小和难度：

| 地图类型 | 总怪物数 | BOSS数 | 精英数 | 普通怪 |
|----------|----------|--------|--------|--------|
| 新手村（1000x1000） | 15-25 | 0 | 0 | 15-25 |
| 中等地图（1500x1500） | 30-50 | 1-2 | 2-4 | 25-44 |
| 大地图（2000x2000） | 50-80 | 2-3 | 4-6 | 40-71 |
| 副本（小） | 10-20 | 1 | 2-3 | 6-16 |

**密度参考**: 每10000格约2-4个怪物

## ⚠️ 注意事项

### 1. 坐标系

- 使用**游戏坐标系**（Y轴向下增长）
- 原点(0,0)在左上角
- 确保坐标在地图范围内（0 < x < mapWidth, 0 < y < mapHeight）

### 2. 性能考虑

- 单个地图不要超过100个生成点
- 单个生成点的count不要超过20
- spawn_radius过大可能导致怪物分布过于稀疏
- 定期检查并清理不再使用的生成点

### 3. 配置验证

服务端启动时会进行基本验证：

```log
✅ 生成点[1001-东门野鸡群]: 配置正确 → (#1234)
❌ 生成点[1002-错误]: 怪物模板999不存在，跳过
⚠️ 生成点[1003]: 坐标超出地图范围，已修正
```

**常见错误及解决**:

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| `怪物模板XXX不存在` | base_monster_id错误 | 检查monsters.json确认正确的ID |
| `坐标超出地图范围` | x/y过大或为负数 | 调整到合理范围内 |
| `ID重复` | 两个生成点使用了相同ID | 修改为唯一ID |
| `JSON格式错误` | 缺少逗号、引号等 | 使用JSON校验工具检查 |

### 4. Fallback机制

如果配置文件加载失败或某地图没有配置：

```log
⚠️ 地图1: 未找到生成点配置，使用默认算法生成
✅ 地图1算法生成完成: 共27个怪物 (配置了5种)
```

## ❓ 常见问题 FAQ

### Q1: 修改配置后为什么没有立即生效？

**A**: 
- 已存在的怪物不会立即消失或移动
- **新生成的怪物**才会使用新配置
- 如需立即生效：
  1. 重启服务（清空所有怪物重新生成）
  2. 或调用热更新 + 手动清除接口

---

### Q2: 可以让两个生成点重叠吗？

**A**: 
- 技术上可以，但不推荐
- 可能导致同一位置怪物过多
- 玩家体验混乱
- 建议：相邻生成点间距至少20格以上

---

### Q3: count设置为0会怎样？

**A**: 
- 不会生成任何怪物
- 相当于 `is_active: false`
- 但保留配置便于后续启用

---

### Q4: 如何完全移除一个生成点？

**A**: 
- 方法1：设置 `"is_active": false`（推荐，保留配置）
- 方法2：从JSON数组中删除该对象（彻底移除）
- 删除后记得热更新配置

---

### Q5: 配置文件损坏怎么办？

**A**: 
- 服务端有容错机制，不会崩溃
- 会自动Fallback到算法生成
- 从Git恢复备份：
  ```bash
  git checkout HEAD -- Server/GameService/Config/monster_spawns.json
  ```
