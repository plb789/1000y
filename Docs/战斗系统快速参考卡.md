# ⚔️ 战斗系统 - 快速参考卡

## 🎯 一图看懂战斗流程

```
[玩家点击怪物]
    ↓
[选中目标] ──→ [显示信息面板]
    ↓
[检查距离]
    ├── ≤1.5格 → [发起攻击] ──→ [发送WS: 0x1001]
    │                ↓
    │           [服务端计算伤害]
    │                ↓
    │           [返回结果: 0x2001]
    │                ↓
    │           [显示伤害飘字 + 更新血条]
    │                ↓
    │           [怪物AI响应（追击/反击）]
    │
    └── >1.5格 → [自动移动] → [到达后再攻击]
```

## 🔢 关键数值一览

### 攻击相关
```javascript
攻击范围:     1格 (近战)
攻击冷却:     1500ms (普通攻击)
全局CD:       500ms (技能间共享)
命中下限:     30%
命中间限:     95%
```

### 怪物AI
```javascript
更新频率:     500ms/tick
巡逻速度:     3-6秒/点
追击速度:     1-2秒/步 (比巡逻快2-3倍)
逃跑血量阈值: <10% HP
追击范围:
  - 被动型(AIType=0): 2格
  - 主动型(AIType=1): 4格
  - BOSS型(AIType=2): 8格 (永不逃跑)
```

### 伤害公式
```javascript
基础伤害 = 攻击力 - 防御力 * 0.5 + 技能加成
最终伤害 = 基础伤害 * 暴击倍率(1.5) * (1 - 格挡减伤(0.3~0.5)) * 波动(0.9~1.1)

概率判定:
- 暴击率 ≈ 5%
- 闪避率 ≈ 10%
- 格挡率 ≈ 15% (受防御力影响, 最高40%)
```

## 📡 协议速查

### 客户端 → 服务端

| 协议号 | 用途 | 关键字段 |
|--------|------|----------|
| `0x1001` | 普通攻击 | `target_id`, `skill_id`(=0) |

**请求示例**:
```json
{"cmd":"attack","target_id":12345,"skill_id":0}
```

### 服务端 → 客户端

| 协议号 | 用途 | 关键字段 |
|--------|------|----------|
| `0x2001` | 攻击结果 | `damage`, `is_crit`, `is_miss`, `current_hp` |
| `0x2010` | 怪物列表 | `monsters[]` |

**响应示例**:
```json
{
  "cmd": "attack_result",
  "damage": 25,
  "is_crit": false,
  "is_miss": false,
  "is_blocked": true,
  "current_hp": 75,
  "max_hp": 100,
  "is_dead": false
}
```

## 🔄 AI状态转换规则

```
空闲(Idle)
  ↓ [生成巡逻路径]
巡逻(Patrol)
  ↓ [发现敌人(追击范围内)]
警戒(Alert) [等待1秒确认威胁]
  ↓ [敌人在追击范围内]
追击(Chase) [向目标移动]
  ↓ [进入攻击范围]
战斗(Combat) [攻击目标(带冷却)]
  ↓ [目标离开追击范围]
巡逻(Patrol)
  ↓ [自身HP<10% 且非BOSS]
逃跑(Flee) [远离目标]
  ↓ [HP归零]
死亡(Dead) [播放死亡动画]
  ↓ [复活时间到]
复活(Respawn) [恢复满血满位置]
  ↓ [复活完成]
空闲(Idle)
```

**特殊规则**:
- BOSS(AIType=2): 永不进入Flee状态
- 被动型(AIType=0): 不主动进入Alert/Chase（需被攻击）
- 逃跑后如果脱离战斗，回到Patrol而非Chase

## 💡 最佳实践

### 1. 添加新怪物类型
```go
// 1. 在 Config/monsters.json 中配置
{
    "id": 201,
    "name": "新怪物",
    "hp": 200,
    "attack": 15,
    "defense": 8,
    "ai_type": 1,        // 主动型
    "chase_range": 5,     // 追击范围5格
    "respawn_time": 60    // 60秒复活
}

// 2. 在前端BattleSystem.js中添加绘制逻辑（可选）
drawMonster(ctx, monster, x, y, size) {
    if (monster.base_id === 201) {
        // 自定义外观（如三角形）
        ctx.beginPath();
        ctx.moveTo(x, y - size*0.4);
        ctx.lineTo(x - size*0.35, y + size*0.3);
        ctx.lineTo(x + size*0.35, y + size*0.3);
        ctx.closePath();
        ctx.fillStyle = '#00FF00';
        ctx.fill();
        return;
    }
    // ... 默认圆形 ...
}
```

### 2. 添加新技能效果
```go
// 在 battle_service.go 的 PlayerAttackMonster 中：
switch skillID {
case 0:  // 普通攻击
    skillBonus = 0
case 101: // 火球术
    skillBonus = attacker.MagicAttack * 1.5  // 法术加成
    // 触发燃烧Debuff（后续实现）
case 102: // 冰冻术
    skillBonus = attacker.MagicAttack * 1.2
    // 触发减速效果（后续实现）
}

// 前端显示不同特效
if (skillID === 101) {
    this.playFireEffect(targetX, targetY);
} else if (skillID === 102) {
    this.playIceEffect(targetX, targetY);
}
```

### 3. 扩展AI行为
```go
// 在 monster_ai.go 中添加新状态：
const (
    // ... 已有状态 ...
    AIStateStun = 8   // 眩晕
)

func (m *AIMonster) updateStun(now int64) {
    // 被眩晕时不能行动
    if now >= m.StunEndTime {  // 需要新增字段
        m.State = m.PreviousState  // 返回之前的状态
    }
}

// 使用方式：
func (m *AIMonster) ApplyStun(duration int64) {
    m.PreviousState = m.State
    m.State = AIStateStun
    m.StunEndTime = time.Now().UnixMilli() + duration
}
```
