## 🐛 常见问题与解决方案

### 问题1: 怪物不显示
**症状**: 进入地图后看不到任何怪物

**原因分析**:
1. 服务端未推送怪物列表
2. 前端未正确接收/解析消息
3. 怪物生成失败

**解决步骤**:
```bash
# 1. 检查服务端日志
grep "Monster spawned" logs/gameservice.log

# 2. 检查WebSocket消息（F12 Network → WS）
# 应该看到: {"cmd":"monster_list","monsters":[...]}

# 3. 检查前端接收逻辑
# Game.js handleMonsterList() 是否被调用

# 4. 手动触发怪物列表请求
window.GameWS.send(Protocol.CMD_GET_MONSTER_LIST, {map_id: 1});
```

---

### 问题2: 攻击无响应
**症状**: 点击怪物后没有反应，没有伤害数字

**原因分析**:
1. 未选中目标
2. 距离过远但未移动
3. 冷却中
4. WebSocket发送失败
5. 服务端处理异常

**解决步骤**:
```javascript
// 1. 检查是否选中
console.log('selectedTarget:', window.game.battleSystem.selectedTarget);

// 2. 检查距离
const monster = window.game.battleSystem.monsters.get(window.game.battleSystem.selectedTarget);
const dist = Math.hypot(window.game.player.x - monster.x, window.game.player.y - monster.y);
console.log('距离:', dist);

// 3. 检查冷却
console.log('冷却结束时间:', window.game.battleSystem.lastAttackTime + 1500);
console.log('当前时间:', Date.now());

// 4. 检查WebSocket状态
console.log('连接状态:', window.GameWS.isConnected);

// 5. 模拟发送攻击请求
window.GameWS.send(0x1001, {
    cmd: 'attack',
    target_id: monster.id,
    skill_id: 0
});
```

---

### 问题3: 伤害为0或不合理
**症状**: 攻击成功但伤害总是0或负数

**原因分析**:
1. 全部闪避（命中率过低）
2. 格挡减伤过多
3. 攻击力<防御力导致负数（被clamp为0）
4. 配置错误

**解决步骤**:
```go
// 1. 检查命中率配置 (Server/GameService/Battle/battle.go)
hitRate := float64(attacker.Dexterity) / (float64(attacker.Dexterity) + 
          float64(defender.Agility)*0.8 + 50) * 100
// 确保 hitRate >= 30%

// 2. 检查防御力是否过高
fmt.Printf("攻击力: %d, 防御力: %n", attacker.Attack, defender.Defense)
// 建议: 攻击力 > 防防力 * 0.5 才能造成有效伤害

// 3. 临时降低防御力测试
monster.Defense = 0;  // 设置为0测试基础伤害

// 4. 添加详细日志
func CalculateDamage(...) (int, bool) {
    baseDamage := attacker.Attack - int(float64(defender.Defense)*0.5)
    log.Printf("基础伤害计算: %d - %d*0.5 = %d", attacker.Attack, defender.Defense, baseDamage)
    // ...
}
```

### 问题4: 怪物不追击或AI不工作
**症状**: 攻击怪物后它不追击，或者一直原地不动

**原因分析**:
1. AI系统未初始化
2. AI更新循环未启动
3. AIType配置错误（被动型不会主动追击）
4. 追击范围设置过小

**解决步骤**:
```bash
# 1. 检查AI初始化日志
grep "AI system initialized" logs/gameservice.log

# 2. 检查AI更新循环
grep "AI update tick" logs/gameservice.log  # 每500ms应该有一条

# 3. 检查怪物AIType配置
cat Config/monsters.json | grep ai_type
# 0=被动(需靠近才追击), 1=主动, 2=BOSS

# 4. 手动触发AI更新
# 在handler.go中添加测试接口：
func TestAIMonster(w http.ResponseWriter, r *http.Request) {
    monsterSvc := monster.GetService()
    monsterSvc.UpdateAllMonsters(time.Now().UnixMilli())
    w.Write([]byte("AI update triggered"))
}

# 5. 检查OnHurted是否被调用
grep "Monster hurted" logs/gameservice.log
```

### 问题5: 性能问题（卡顿）
**症状**: 怪物多时帧率下降，操作延迟

**原因分析**:
1. 渲染过多怪物（未做视野裁剪）
2. 伤害飘字对象未回收
3. AI更新频率过高
4. 内存泄漏

**解决步骤**:
```javascript
// 1. 检查视野裁剪是否生效
// BattleSystem.js render() 方法应该包含：
if (screenX < -tileSize || screenX > ctx.canvas.width + tileSize ||
    screenY < -tileSize || screenY > ctx.canvas.height + tileSize) {
    return;  // 不渲染屏幕外的怪物
}

// 2. 检查伤害飘字清理
// renderDamageNumbers() 应该过滤过期飘字：
this.damageNumbers = this.damageNumbers.filter(d => !isExpired);

// 3. 降低AI更新频率
// monster_ai.go UpdateAllMonsters() 中：
if now-lastUpdate < 500 { return }  // 至少500ms更新一次

// 4. 监控内存使用
// Chrome DevTools → Performance → Memory
// 录制30秒，检查内存曲线是否持续上升
```
