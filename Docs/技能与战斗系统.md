## 2. 战斗伤害计算

### 2.1 技能伤害公式

```
基础伤害 = skill.Damage                    // 技能配置的基础伤害
        + (攻击力 × 30%)                   // 攻击力转化比例
        + (等级-1) × skill.Damage × 5%      // 每级成长+5%
```

### 2.2 完整伤害流程

```
① 命中判定 → CalculateHit()
   └─ 失败 → 闪避(damage=0, isMiss=true)

② 格挡判定 → CalculateBlock()
   └─ 成功 → damage × (1 - blockReduction), isBlocked=true

③ 暴击判定 → Crit% vs rand(0,100)
   └─ 成功 → damage × (CritDamage / 100), isCrit=true

④ BUFF减伤 → GetTotalDamageReduction(targetID)
   └─ reductionPct != 0 → damage × (1 - reductionPct/100)
   └─ reductionPct >= 100 → 无敌免疫(damage=0, invulnerable=true)

⑤ 伤害波动 → ±10% 随机浮动

⑥ 反弹伤害 → GetTotalReflectPct(targetID)
   └─ reflectPct > 0 → 攻击者受到 damage×reflectPct% 伤害

⑦ 吸血回复 → GetTotalLifestealPct(attackerID)
   └─ lifestealPct > 0 → 攻击者恢复 damage×lifestealPct% HP

⑧ 应用伤害到目标 → MonsterTakeDamage()

⑨ 目标DEBUFF附加(概率触发, 见 §3)

⑩ 自身BUFF附加(内功/身法/护体, 100%触发, 见 §5.3)

⑪ AOE范围伤害(如果 aoe_radius > 0)
```

### 2.3 普通攻击公式（非技能）

```
damage = (攻击力 - 防御力/2) × SkillDamageFactor[type] × (1 + level×5%)
```

### 2.4 属性加成来源（叠加关系）

```
最终属性 = 角色基础值
         + 武学技能加成(CalculateSkillBonus)
         + BUFF加成(CalculateEffect)
         + 装备加成(待接入)
```

## 6. BUFF定时Tick系统

> 实现日期: 2026-06-23 | 状态: ✅ 已完成并编译通过

### 6.1 系统概述

**功能**: 每秒自动触发所有持续效果型BUFF的HP/MP变化（如中毒掉血、打坐回血、回蓝等）

**架构设计**:
```
buff_manager.go (定时器驱动)
    ↓ 每秒调用 tickAllTargets()
Handler (BuffTickApplier接口实现)
    ├── ApplyPlayerHPChange() → DB服务修改玩家HP
    ├── ApplyPlayerMPChange() → DB服务修改玩家MP
    ├── ApplyMonsterHPChange() → 怪物服务修改怪物HP(含死亡检测)
    └── BroadcastBuffTickBatch() → Gateway WebSocket → 前端显示
```

### 6.2 核心组件

#### 6.2.1 BuffTickApplier 接口

**文件**: `Server/GameService/Buff/buff_manager.go:367-384`

```go
type BuffTickApplier interface {
    ApplyPlayerHPChange(playerID uint64, hpChange int)           // 修改玩家HP
    ApplyPlayerMPChange(playerID uint64, mpChange int)           // 修改玩家MP
    ApplyMonsterHPChange(monsterID uint64, hpChange int) (int, bool) // 修改怪物HP
    BroadcastBuffTick(targetID uint64, targetType uint8, hpChange int, mpChange int) // 单个广播
    BroadcastBuffTickBatch(results []BuffTickResult)             // 批量广播(优化)
}
```

#### 6.2.2 BuffTickResult 数据结构

```go
type BuffTickResult struct {
    TargetID   uint64  // 目标ID
    TargetType uint8   // 1=玩家, 2=怪物
    HpChange   int     // HP变化(正=恢复, 负=伤害)
    MpChange   int     // MP变化(正=恢复, 负=消耗)
}
```

#### 6.2.3 定时器管理

**启动入口**: `Server/GameService/main.go:397-400`

```go
buffManager := buff.GetManager()
buffManager.StartTicker(battleHandler)  // 注入Handler作为接口实现
```

**定时器行为**:
- **间隔**: 固定1秒 (`time.NewTicker(1 * time.Second)`)
- **生命周期**: 随GameService启动/停止
- **重复启动保护**: 检测`globalTicker != nil`则跳过
- **清理机制**: 每60秒调用`CleanupExpired()`清理过期BUFF

### 6.3 Tick处理流程

#### 6.3.1 后端处理流程

```
[每秒触发]
  ↓
tickAllTargets(applier)
  │
  ├── 1. 快照所有有BUFF的targetID (读锁,避免阻塞)
  │
  ├── 2. 遍历每个目标:
  │   │
  │   ├── m.Tick(targetID, now) → 计算本次HP/MP变化
  │   │   └── 过滤: hpChange==0 && mpChange==0 则跳过
  │   │
  │   ├── 判断目标类型 (targetType):
  │   │   ├── type=1 (玩家):
  │   │   │   ├── applier.ApplyPlayerHPChange(id, hp) → DBRoleChangeHP()
  │   │   │   └── applier.ApplyPlayerMPChange(id, mp) → DBRoleChangeMP()
  │   │   │
  │   │   └── type=2 (怪物):
  │   │       ├── applier.ApplyMonsterHPChange(id, hp) → MonsterTakeDamage()
  │   │       └── if isDead:
  │   │           ├── log.Printf("怪物被BUFF击杀")
  │   │           └── m.ClearAllBuffs(targetID)
  │   │
  │   └── 收集结果到 []BuffTickResult
  │
  ├── 3. 批量广播: applier.BroadcastBuffTickBatch(results)
  │   └── 按地图ID分组 → 减少HTTP请求次数
  │
  └── 4. 定期清理: now.Unix()%60 == 0 → CleanupExpired()
```

#### 6.3.2 批量广播优化

**问题**: N个目标有BUFF → N次HTTP POST到Gateway

**解决方案**: 按地图ID分组后批量发送

| 场景 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 10个怪物在2个地图中毒 | 10次请求 | 2次请求 | **80%↓** |
| 20个玩家打坐 | 20次请求 | 1次请求(同地图) | **95%↓** |

**实现代码**: `Server/GameService/Battle/handler.go:612-677`

```go
func (h *Handler) BroadcastBuffTickBatch(results []buff.BuffTickResult) {
    mapBatches := make(map[uint32]*MapBatch)  // 按地图分组
    
    for _, result := range results {
        // 获取目标所在地图ID...
        mapBatches[mapID].Ticks = append(mapBatches[mapID].Ticks, tickMsg)
    }
    
    // 批量广播每个地图
    for mapID, batch := range mapBatches {
        h.broadcastToMap(mapID, "buff_tick_batch", batchMsg)
    }
}
```

```javascript
    // 绘制四边半透明边框(上/下/左/右各20px宽)
    // 渐隐效果: alpha *= (1 - progress)
}
```

**参数说明**:
- `color`: 闪烁颜色（默认红色）
- `duration`: 持续时间毫秒（默认300ms）
- `alpha`: 最大透明度（默认0.3）

#### 6.4.4 怪物被BUFF击死特效序列

```
时间轴:
  0ms   - 服务端推送 buff_tick (hp_change=-20, 导致hp≤0)
        ↓
  0ms   - 前端 handleBuffTick():
          · monster.hp = Math.max(0, monster.hp + hpChange)
          · monster.status = 4 (死亡状态)
          · triggerDeathAnimation(monsterId)  // 淡出+放大1.5s
          · showFloatingText("BUFF击杀 史莱姆", x, y-1.5, '#FF00FF')
        ↓
  0ms   - 显示紫色 "-20 HP" 飘字
        ↓
  3000ms - setTimeout → removeMonster(targetId)
           (等待死亡动画完成后移除)
```

### 6.6 配置文件影响

BUFF的持续效果完全由 `buffs.json` 配置决定：

**关键字段**:

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `hp_change` | int | 每秒HP变化 | 50(回血), -20(中毒), 0(无) |
| `mp_change` | int | 每秒MP变化 | 10(回蓝), -5(耗蓝), 0(无) |
| `duration` | int | 持续时间(秒) | 0(永久), 300(5分钟), 10(短DEBUFF) |

**注意事项**:
- `duration=0` 表示永久BUFF（如打坐），需手动取消或角色下线清理
- 正负数值决定效果方向：正数=增益，负数=减益
- 叠加层数会影响最终数值：`实际变化 = config值 × stack`
