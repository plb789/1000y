# UI编辑器使用说明

## 简介

这是一个专门为"千年江湖"游戏客户端设计的可视化UI编辑器，可以实时编辑客户端的UI界面，支持元素选择、拖拽移动、调整大小、样式编辑、图片资源管理等功能。

## 功能特点

### 1. 实时编辑
- **所见即所得**：直接在客户端UI上进行编辑，实时预览效果
- **元素选择**：点击预览区的任意元素即可选中
- **拖拽移动**：选中元素后可以拖拽移动位置
- **调整大小**：通过调整手柄改变元素尺寸

### 2. 场景管理
- **登录界面**：编辑登录页面的UI布局
- **角色选择**：编辑角色选择界面
- **创建角色**：编辑角色创建界面
- **游戏主界面**：编辑游戏内的UI框架

### 3. 样式编辑
- **背景设置**：背景色、透明度、背景图片
- **边框设置**：边框颜色、宽度、圆角
- **文字设置**：文字颜色、字号、对齐方式
- **效果设置**：阴影效果

### 4. 布局编辑
- **定位方式**：静态、相对、绝对、固定定位
- **尺寸设置**：宽度、高度
- **间距设置**：内边距、外边距
- **层级控制**：Z-Index、显示方式、可见性

### 5. 图片资源管理
- **上传图片**：支持拖拽或点击上传图片
- **图片应用**：将图片应用到选中元素的背景或图片标签
- **预设背景**：提供多种渐变背景模板
- **持久化存储**：上传的图片会保存到本地存储

### 6. 新增组件
- **面板容器**：可自定义的容器面板
- **按钮**：可点击的按钮组件
- **输入框**：文本输入框
- **文本**：文本显示组件
- **图片**：图片显示组件
- **状态栏**：顶部或底部状态栏
- **进度条**：血条、蓝条等进度条
- **图标**：图标或徽章组件

### 7. 编辑操作
- **复制/粘贴**：Ctrl+C复制，Ctrl+V粘贴
- **撤销/重做**：Ctrl+Z撤销，Ctrl+Y重做
- **删除**：Delete键删除选中元素
- **保存**：Ctrl+S保存修改

### 8. 导出功能
- **导出HTML**：导出完整的HTML文件
- **导出CSS**：导出样式代码
- **导出JSON**：导出修改配置
- **应用到客户端**：直接导出并替换客户端文件

## 使用步骤

### 第一步：打开编辑器
1. 打开浏览器，访问 `Frontend/UIEditor/index_fixed.html`
2. 编辑器会显示一个加载界面，提示您选择客户端HTML文件

### 第二步：加载客户端HTML
有两种方式加载客户端HTML：

**方式一：选择文件**
1. 点击"选择文件"按钮
2. 选择 `Frontend/index.html` 文件
3. 编辑器会自动加载并显示客户端UI

**方式二：手动粘贴**
1. 打开 `Frontend/index.html` 文件
2. 复制全部内容
3. 在编辑器的文本框中粘贴HTML内容
4. 点击"加载HTML"按钮

### 第二步：开启编辑模式
1. 点击顶部工具栏的"开启编辑"按钮
2. 编辑模式开启后，可以点击预览区的元素进行选择

### 第三步：选择场景
1. 在左侧面板选择要编辑的场景（登录、角色选择、创建角色、游戏主界面）
2. 或使用顶部下拉框切换场景

### 第四步：编辑元素
1. **选择元素**：点击预览区的元素，或点击左侧组件树的节点
2. **查看属性**：右侧面板会显示选中元素的属性
3. **修改样式**：在样式面板修改背景、边框、文字等属性
4. **修改布局**：在布局面板修改定位、尺寸、间距等属性
5. **拖拽移动**：选中元素后，直接拖拽移动位置
6. **调整大小**：通过元素周围的调整手柄改变尺寸

### 第五步：应用图片
1. 切换到"图片"标签页
2. 点击上传区域上传图片
3. 选择要应用图片的元素
4. 点击已上传的图片，图片会自动应用到选中元素

### 第六步：新增组件
1. 点击"+ 新增组件"按钮
2. 选择要添加的组件类型
3. 新组件会添加到当前场景中
4. 可以继续编辑新组件的属性

### 第七步：保存修改
1. 点击"保存"按钮，保存到本地存储
2. 或点击"导出代码"，导出HTML/CSS/JSON文件
3. 或点击"应用到客户端"，导出HTML文件并替换客户端文件

## 注意事项

### 1. 关键元素保护
以下元素不能删除：
- loginPanel（登录面板）
- roleSelectPanel（角色选择面板）
- roleCreatePanel（角色创建面板）
- gamePanel（游戏主面板）
- gameCanvas（游戏画布）
- topBar（顶部状态栏）
- bottomBar（底部快捷栏）

### 2. 图片资源
- 上传的图片会保存在浏览器的本地存储中
- 清除浏览器数据会导致图片资源丢失
- 建议定期导出修改，避免数据丢失

### 3. 应用到客户端
- 点击"应用到客户端"会下载修改后的HTML文件
- 需要手动将下载的文件放入 `Frontend` 目录，替换原有的 `index.html` 文件
- 替换前建议备份原文件

### 4. 浏览器兼容性
- 推荐使用 Chrome、Edge、Firefox 等现代浏览器
- 需要支持 localStorage 和 postMessage API

## 技术架构

### 文件结构
```
Frontend/UIEditor/
├── index_fixed.html     # 编辑器主文件（推荐使用）
├── index_new.html       # 编辑器主文件（旧版本，需要服务器）
├── editor-inject.js     # 编辑器注入脚本
└── README.md            # 使用说明文档
```

**重要说明**：
- `index_fixed.html` 是改进版本，可以在本地文件系统直接使用
- `index_new.html` 是旧版本，需要通过HTTP服务器访问才能正常工作
- 推荐使用 `index_fixed.html`

### 通信机制
编辑器通过 `postMessage` API 与客户端页面通信：
- **编辑器主文件**：运行在编辑器窗口
- **注入脚本**：注入到客户端页面，负责实际的DOM操作
- **消息通道**：通过 postMessage 实现双向通信

### 核心功能
1. **元素选择**：通过点击获取元素信息
2. **拖拽移动**：监听鼠标事件实现拖拽
3. **调整大小**：通过调整手柄改变元素尺寸
4. **样式编辑**：通过消息更新元素样式
5. **撤销重做**：保存历史状态实现撤销重做

## 常见问题

### Q: 编辑器无法连接到客户端？
A: 
1. 检查浏览器是否支持 postMessage
2. 确保客户端页面已完全加载
3. 刷新编辑器页面重试

### Q: 点击元素无法选中？
A: 
1. 确认编辑模式已开启（按钮显示"关闭编辑"）
2. 检查连接状态是否显示"已连接"
3. 刷新页面重试

### Q: 修改后没有生效？
A: 
1. 点击"保存"按钮保存修改
2. 点击"应用到客户端"导出文件
3. 将导出的文件替换客户端原文件

### Q: 图片上传失败？
A: 
1. 检查图片格式是否支持（支持常见图片格式）
2. 检查浏览器是否有足够的存储空间
3. 尝试上传较小的图片
