## 一、项目概述
### 1.1 项目简介
《千年江湖》是基于经典千年武侠框架重构开发的**复古2D武侠MMORPG全栈网游**，保留原版经典江湖玩法、武学系统、门派体系、PK竞技、野外探索核心体验，同时完成架构现代化升级。
项目采用**Electron客户端（HTML5 Canvas前端）+ Go微服务（WebSocket）+ MySQL 8.0** 全栈技术体系，支持跨端客户端登录、多人在线实时联机、跨场景交互、后台运营管理、数据实时统计，兼顾复古游戏体验与现代服务端稳定性、扩展性，可支持千人同时在线，适用于私服运营、局域网联机、定制化二次开发。

### 1.2 项目目标
1. 复刻经典千年核心武侠玩法，还原打怪、练功、武学、门派、交易、攻城等经典模块；
2. 摒弃老旧C++框架，基于Go+HTML5 Canvas重构全栈体系，降低开发与维护成本；
3. 搭建高可用分布式微服务架构，支持多服、分区、合服、热更新；
4. 配套完整Web运营后台、GM工具、日志监控、数据统计系统；
5. 提供规范源码结构，支持功能迭代、模组开发、皮肤/场景/道具自定义扩展。

### 1.3 适用人群
开发人员、运维人员、运营管理人员、二次开发开发者、游戏私服运维团队。

## 二、整体技术架构
### 2.1 架构总览
项目采用**微服务拆分 + 前后端分离**模式，整体分为：**Electron前端客户端层、Go微服务服务端层、Web运营后台层**，各层通过网络协议解耦，独立部署、互不影响。
整体架构拓扑：
`Electron客户端 <--> 网关微服务(WebSocket) <--> 登录/游戏/数据微服务 <--> 数据存储层`
`Web后台/GM工具 <--> Nginx反向代理 <--> 接口微服务 <--> 数据库/缓存`

### 2.2 技术栈明细
#### 2.2.1 前端（Electron客户端）
- 核心技术：**原生HTML + Canvas 2D + JavaScript（无框架）**
- 运行载体：Electron（跨Windows/macOS/Linux，保留客户端体验）
- 渲染能力：Canvas 2D渲染引擎（解析千年官方.map二进制瓦片地图、角色帧动画、场景特效）
- 网络通信：WebSocket长连接（基于gorilla/websocket客户端实现，实时联机、消息广播）
- 资源格式：兼容原版千年资源（DDS贴图、SPR帧动画、MAP场景地图、INI配置）
- 性能优化：轻量无框架设计，原生JS高性能渲染，瓦片地图分片加载
- 附加能力：全屏/窗口化切换、本地缓存、离线数据暂存、快捷键自定义

#### 2.2.2 服务端技术栈（核心逻辑）
1. **网关微服务**
   - 语言：Go 1.22
   - 核心依赖：gorilla/websocket
   - 功能：客户端WebSocket连接接入、协议转发、连接管理、流量过滤、防攻击、会话保持、多服路由
2. **登录微服务**
   - 语言：Go 1.22
   - 功能：账号校验、登录授权、分区选择、Token发放、防重复登录、IP封禁
3. **游戏微服务**
   - 语言：Go 1.22
   - 功能：角色逻辑、战斗、武学、NPC、怪物、场景、任务、交易、PK、门派核心业务、实时广播
4. **数据DB微服务**
   - 语言：Go 1.22
   - 功能：数据库CRUD封装、读写分离、缓存管理、数据持久化、事务控制
5. **数据存储**
   - 主数据库：**MySQL 8.0（主从集群）** 存储角色、道具、账号、日志、配置等持久化数据
   - 缓存：Redis 6+ 存储在线玩家、临时状态、排行榜、高频读取数据
   - 日志存储：本地文本 + 日志归集，支持按天分割

#### 2.2.3 Web运营后台 & GM工具
- 前端框架：**Vue3 + Vite + Element Plus + Sass**（可选，也可复用原生HTML/JS轻量化实现）
- 后端接口：Go 1.22（Gin + GORM）RESTful API
- 部署：**Nginx** 反向代理、静态资源托管、接口转发
- 功能：账号管理、角色查询、道具发放、GM指令、服务器状态监控、数据统计、公告发布、封号/解封、配置在线修改

#### 2.2.4 运维&部署工具
- 版本管理：Git / TortoiseGit
- 容器化（可选）：Docker 打包服务端、数据库、中间件
- 服务器系统：CentOS 7 / Ubuntu 20.04（服务端）、跨平台（Electron客户端）
- 反向代理：Nginx（前端静态资源、接口转发、负载均衡）
- 监控：服务状态自检、在线人数监控、异常连接告警

### 2.3 网络协议设计
1. **前端 ↔ 服务端**：WebSocket长连接（JSON/二进制协议可选），数据包加校验码、简单加密，防止封包篡改、外挂抓包；
2. **服务端内部**：微服务之间基于gRPC/HTTP + RabbitMQ异步通信；
3. **Web后台 ↔ 服务端**：标准HTTP/HTTPS + Token身份认证接口，Nginx反向代理转发。

### 2.4 部署架构（标准生产环境）
```
外网 → 防火墙 → Nginx(Web后台/静态资源/接口转发)
              ↓
          网关微服务(WebSocket连接接入)
              ↓
      登录微服务 → 游戏微服务 → 数据DB微服务
              ↓
      Redis集群 + MySQL主从数据库
```

## 三、项目核心特色
### 3.1 玩法特色
1. **经典复古武侠体系**
   完整复刻千年原版**武学修炼、内功、外功、身法、护体**系统，支持手动练功、挂机练功、武学进阶，还原武侠修炼核心乐趣。
2. **自由PK与江湖对抗**
   野外自由PK、门派对战、城战/攻城战、擂台比武，支持善恶值、红名、通缉、仇家记录机制。
3. **门派与社交系统**
   多门派选择、门派职位、门派仓库、门派任务、门派聊天、师徒系统、好友、组队、公会。
4. **经济与交易系统**
   玩家面对面交易、摆摊、拍卖行、货币体系（金币/绑定金币/元宝）、道具掉落、材料合成、装备强化。
5. **大世界探索**
   多张经典武侠场景、野外怪物、BOSS、副本、秘境、限时活动、地图传送、驿站赶路。

### 3.2 技术架构特色
1. **全栈Go+HTML5重构**
   摒弃老旧C++框架，基于Go微服务+HTML5 Canvas原生渲染，跨端兼容、开发效率高、维护成本低。
2. **WebSocket实时联机**
   基于gorilla/websocket实现低延迟长连接，支持实时战斗、场景同步、全服广播，满足MMORPG联机需求。
3. **微服务解耦设计**
   网关、登录、游戏、数据微服务独立部署，单一模块故障不影响整体服务，支持弹性扩缩容。
4. **原生解析千年地图**
   Canvas 2D原生解析千年官方.map二进制瓦片地图，完美还原经典场景，兼容原版资源。
5. **高并发支持**
   网关微服务连接池、Redis热点数据缓存、MySQL主从读写分离，支持**千人级同时在线**。
6. **安全防护**
   WebSocket数据包加密、IP限制、恶意连接拦截、GM权限分级、操作日志留痕。
7. **易扩展二次开发**
   标准化源码目录、模块化编码、配置文件外置，支持新增地图、道具、武学、任务、活动，无需改动核心底层。
8. **全链路监控**
   在线人数、服务器负载、数据库压力、异常报错、玩家行为日志全记录，运维可视化。

### 3.3 运营特色
1. 可视化Web后台，零代码实现发道具、改角色、控服务器；
2. 在线配置修改，无需重启服务器即可更新活动、怪物、掉落、商城；
3. 完整数据统计：在线曲线、道具产出、金币流水、活跃玩家分析；
4. 全自动日志分割、数据备份、防数据丢失；
5. Nginx反向代理加持，静态资源加载更快、接口转发更稳定。

## 六、数据库设计简述（核心表）
数据库采用MySQL主从架构，核心数据表如下：
1. **account**：账号表（账号、密码、注册时间、状态、IP）
2. **role**：角色主表（角色ID、账号ID、等级、属性、门派、善恶值）
3. **role_item**：角色道具表（背包、仓库道具、数量、有效期）
4. **role_skill**：角色武体表（武学ID、熟练度、等级）
5. **role_buff**：角色临时Buff表
6. **friend**：好友/黑名单表
7. **team**：组队信息表
8. **faction**：门派主表、门派成员表
9. **mail**：邮件表、邮件附件表
10. **trade_record**：交易流水、摆摊记录、拍卖行记录
11. **game_log**：全服行为日志（登录、下线、掉落、交易、GM操作）
12. **server_config**：服务器动态配置表

## 七、编译、启动与部署说明（简要）
1. **前端编译打包**：安装Electron依赖，执行 `npm run build` 打包生成跨端客户端；
2. **服务端编译**：Go微服务执行 `go build` 编译各微服务可执行文件；
3. **数据库初始化**：执行 `Script/SQL/init.sql` 创建数据表与初始数据；
4. **中间件部署**：安装 MySQL、Redis、Nginx，修改对应配置文件；
5. **Nginx配置**：配置反向代理转发Web后台、前端静态资源、接口请求；
6. **启动顺序**：数据库 → Redis → Nginx → 网关微服务 → 登录微服务 → 游戏微服务 → 数据DB微服务 → Web后台 → Electron客户端；
7. **运维**：使用守护进程脚本保活Go微服务，定时执行数据备份。

## 八、二次开发规范
1. 所有新增功能遵循**模块化开发**，不篡改核心底层代码；
2. 新增WebSocket协议统一在 `Protocol` 目录定义，版本向后兼容；
3. 配置类数据全部外置到 `Config` 目录，禁止硬编码数值；
4. Canvas渲染新增特效/地图需兼容原版.map格式，保证加载性能；
5. 所有新增接口、功能必须编写日志，便于问题排查；
6. Go微服务新增接口需遵循RESTful规范，WebSocket消息格式统一。
